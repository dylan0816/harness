# Context Guard

> 优先级：辅助行为，不干扰主任务，沉淀失败不阻塞任务完成。

## 触发条件

1. **主任务收尾阶段**（任务完成前，最后一步执行）
2. **用户显式发起**（`#guard`、`#guard force`、`#guard skip`、`#guard show`）

## 价值判定

AI 基于对话事实（非主观陈述）逐项核对以下信号：

1. 代码/配置出现结构变更（新增文件/接口/模块） → 架构决策
2. 持续报错或行为不符合预期，最终定位根因并修复 → 根因分析
3. 建立新的命名规则、错误码格式、接口约定或 lint 规则 → 规范确立
4. 修复中纠正了 AI 或用户对库/语言特性/框架行为的错误理解 → 知识盲区
5. 比较两种以上方案，给出优劣依据并选择其一 → 方案对比
6. 解决不直接报错但破坏构建/运行的问题 → 踩坑记录

**结果**：命中 ≥1 个 → 询问用户；全部未命中 → 跳过，回复"已评估知识沉淀价值（跳过：{理由}）"，必须列出检查了哪些信号。

**不触发**：纯执行性操作（改变量名/加注释/修 typo）、简单问答（函数用法/语法确认/配置路径）。

## 执行流程

### A. 询问

> 本次对话涉及 **[信号名]**，建议记录要点。是否记录？（Y/N/补充）

用户 N 或模糊 → 跳过。用户否认清晰命中 → 尊重决定。用户要求自动记录 → 拒绝（见边界）。用户确认后执行下一步。

### B. 摘要

```
## YYYY-MM-DD
**主题**: [一句话]
**类型**: #architecture | #troubleshooting | #convention | #knowledge | #decision
**背景**: [触发讨论的代码/配置/行为]
**结论**: [做了什么、为什么、不再做什么]
**信号**: [命中的信号名]
```

只写结论，不写流水，≤300 字。信号为必填。

### C. 写入

`<工程根>/.context-guard/YYYY-MM-DD.md`。目录不存在则创建。当日文件已存在则追加到末尾。不合并。

## 与 key-points-mem 的协作

| | key-points-mem | context-guard |
|---|---|---|
| 位置 | `.claude/skills/.../references/` | `<工程根>/.context-guard/` |
| 内容 | 精炼知识点（现象-原因-解决-经验） | 对话上下文+结论，按天归档 |
| 范围 | 跨工程通用 | 工程专属 |
| 触发 | 用户授权后 AI 提炼 | 自动评估 + 用户确认 |

两个系统独立运行，互不查询。提炼由用户手动发起，AI 不做自动计划。

## 边界

1. **不阻塞主任务**：收尾阶段执行，不打断主流程。
2. **不自动写入**：必须经用户确认。用户要求"自动记"时拒绝。
3. **上下文保护**：预估剩余不足 500 token 时跳过。
4. **工程隔离**：不跨工程读/写。跨天对话只写入最后一天文件。
5. **清理**：用户自行决定，AI 不主动提议。

## 任务结束标志

Stop hook 检测本轮文件变更后强制要求评估。AI 无需预设格式——hook 阻断时会告知具体写法。

## 快速触发

`#guard` 执行判定 / `#guard force` 跳过判定直接询问 / `#guard skip` 跳过本次 / `#guard show` 查看当日记录。
